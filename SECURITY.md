# Security Policy

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| 1.0.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in Sleep Logger, please report it responsibly.

### How to Report

**Preferred Method:**
- Email: security@sleeplogger.app
- Use PGP key if available (provided on request)

**Alternative Method:**
- Create a private issue on GitHub
- Mark with "security" label

### What to Include

- Detailed description of the vulnerability
- Steps to reproduce (if applicable)
- Potential impact assessment
- Any proof-of-concept code

### Response Timeline

- **Initial response**: Within 48 hours
- **Detailed assessment**: Within 7 days
- **Fix deployment**: As soon as possible, based on severity

### Security Model

Sleep Logger is designed with privacy in mind:

- **Local Storage**: All data stored locally on device
- **No Network**: No internet connectivity for data
- **No Permissions**: Minimal app permissions
- **Open Source**: Code available for security review

### Security Features

- Data encryption at rest (Android keystore)
- No third-party analytics
- No user tracking
- Minimal permission requirements

### Out of Scope

The following are not considered security vulnerabilities:

- Issues requiring physical device access
- Issues on rooted devices
- Issues from modified app versions
- General usability issues

### Recognition

Security researchers who responsibly disclose vulnerabilities will be:
- Acknowledged in release notes
- Listed in our Hall of Fame
- Eligible for Sleep Logger merchandise

### Legal

We commit to:
- Not pursue legal action against researchers
- Keep disclosures confidential
- Work with researchers to address issues

Thank you for helping keep Sleep Logger secure! 🔒
